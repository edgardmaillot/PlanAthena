Chantier? chantierDomaine = null )

Valide l'ensemble des données d'entrée du chantier. Cette méthode englobe :

  1. La validation syntaxique et structurelle des DTOs (P2 - souvent déléguée à FluentValidation en amont).
  2. La validation des références croisées (P3 - ex: un TacheDto.BlocId existe bien).
  3. La détection de cycles de dépendances (P4 - ex: cycles entre TacheDto, cycles entre LotTravauxDto).
Parameters
inputDtoLes données d'entrée complètes du chantier.
chantierDomaineOptionnel: L'objet Chantier du domaine déjà mappé, si certaines validations sont plus faciles à effectuer sur le modèle de domaine.
Returns
Une liste de messages de validation (erreurs ou avertissements).
